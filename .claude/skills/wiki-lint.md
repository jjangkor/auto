# wiki-lint

위키 건강 상태를 점검하고 유지보수합니다.

## 사용법
`/wiki-lint`

## 점검 항목

1. **모순 검출**: 페이지 간 충돌하는 정보가 있는지 확인
2. **스테일 정보**: 최신 소스로 대체되어야 할 오래된 내용
3. **고아 페이지**: 다른 페이지에서 링크되지 않는 페이지
4. **누락 페이지**: `[[위키링크]]`가 있지만 실제 페이지가 없는 것
5. **누락 크로스레퍼런스**: 관련성 높은데 링크가 없는 페이지 쌍
6. **인덱스 동기화**: index.md가 실제 위키 구조와 일치하는지
7. **index_summary 최신성**: 요약이 현재 위키 상태를 반영하는지
8. **frontmatter 일관성**: YAML frontmatter 형식 준수 여부
9. **지식 갭**: 다뤄야 할 중요 주제인데 페이지가 없는 것
10. **추가 소스 제안**: 위키를 보강할 수 있는 새 소스 방향

## 워크플로우

1. `wiki/index.md` 읽기
2. 전체 위키 페이지 스캔 (Glob으로 파일 목록 → 각 페이지 읽기)
3. 위 점검 항목별로 이슈 수집
4. 사용자에게 결과 보고 (심각도순 정렬)
5. 사용자 승인 후 수정 작업 진행
6. `wiki/log.md` 에 lint 결과 기록

## 출력 형식

```
### Wiki Lint Report - YYYY-MM-DD

**요약**: 총 N개 이슈 발견

#### Critical
- [ ] 이슈 설명...

#### Warning  
- [ ] 이슈 설명...

#### Suggestion
- [ ] 이슈 설명...
```
