# wiki-ingest

소스 파일을 읽고 위키에 정리합니다.

## 사용법
`/wiki-ingest [소스 경로]`

예시:
- `/wiki-ingest raw/article.md` (단일 파일)
- `/wiki-ingest raw/books/` (폴더 내 미처리 파일 전부)
- `/wiki-ingest /home/user/notes/` (외부 폴더)
- `/wiki-ingest` (경로 미지정 시 사용자에게 확인)

## 워크플로우

1. **소스 확인**: 지정된 경로의 md 파일을 읽습니다. 디렉토리가 지정되면 하위 전체를 스캔하여 아직 인제스트되지 않은 파일을 찾습니다 (wiki/log.md와 대조). 경로가 없으면 사용자에게 확인합니다.

2. **핵심 추출**: 소스를 읽고 핵심 내용을 파악합니다:
   - 주요 주장/사실
   - 등장하는 엔티티 (인물, 조직, 장소 등)
   - 핵심 개념/프레임워크
   - 기존 위키 내용과의 관계 (보강, 모순, 새로운 관점)

3. **사용자와 논의**: 핵심 테이크어웨이를 사용자에게 보여주고, 어떤 부분을 강조할지 의견을 물어봅니다.

4. **위키 페이지 생성/수정**:
   - `wiki/sources/` 에 소스 요약 페이지 생성
   - 새 엔티티 → `wiki/entities/` 에 페이지 생성
   - 새 개념 → `wiki/concepts/` 에 페이지 생성
   - 기존 페이지에 새 정보 통합 (단순 추가가 아닌 합성)
   - 모순 발견 시 양쪽 페이지에 명시

5. **크로스레퍼런스**: 생성/수정된 모든 페이지 간 `[[위키링크]]` 연결 확인

6. **인덱스 업데이트**:
   - `wiki/index.md` 에 새 페이지 추가, 기존 항목 수정
   - `wiki/index_summary.md` 필요시 업데이트

7. **로그 기록**: `wiki/log.md` 에 인제스트 기록 append (소스 경로 포함)

## 주의사항
- 소스 파일은 절대 수정하지 않음 (어떤 경로든)
- 한 소스에서 여러 페이지를 터치하는 것이 정상 (10-15개도 가능)
- YAML frontmatter 형식 준수 (CLAUDE.md 참조)
- 이미지가 포함된 소스는 텍스트 먼저 읽고, 이미지를 별도로 확인
- 디렉토리 지정 시 log.md의 기존 인제스트 기록과 대조하여 미처리 파일만 처리
- 정리본은 항상 `wiki/` 하위에 생성 (소스가 어디에 있든)
